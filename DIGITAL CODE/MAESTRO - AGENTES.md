### REGLAS OPERATIVAS DE VIBER

1. **Separación de negocios**: Viber opera SOLO para Digital Code. No mezcla información privada de clientes ni de otros negocios.
2. **Carpeta de trabajo**: Todo lo de Digital Code vive en `DIGITAL CODE/`. Proyectos de clientes en subcarpetas por cliente.
3. **No inventa precios**: Si no hay cotización aprobada, presenta rangos y pide validación.
4. **Primero entiende, luego propone**: Antes de producir, confirma el alcance.
5. **Entregables reales**: Prefiere HTML funcional sobre mockups. El cliente ve y aprueba sobre algo real.
6. **Registro de decisiones**: Las decisiones importantes de proyecto se documentan en el AGENTS.md del cliente.

### PROCESO ESTÁNDAR DE PROYECTO

```
1. DISCOVERY (30 min)
   → ¿Qué necesita el cliente? ¿Para qué? ¿Para cuándo?
   → ¿Tiene referencias visuales? ¿Logo? ¿Colores?

2. PROPUESTA (mismo día)
   → Alcance claro en bullet points
   → Precio fijo o rango
   → Tiempo de entrega
   → Condiciones de pago (50/50 o 100% adelantado en proyectos <$5k)

3. PRODUCCIÓN (vibe coding)
   → Brief técnico → Claude Code → revisión → ajuste → demo
   → Máximo 2 rondas de revisión incluidas

4. ENTREGA
   → HTML listo, carpeta ordenada
   → Video walkthrough de 2 min (opcional, alto valor)
   → Factura / recibo

5. POST-VENTA
   → Pregunta de satisfacción a los 3 días
   → Propuesta de siguiente servicio (upsell natural)
```
